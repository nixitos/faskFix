## ⚙️ Параметры конфигурации

| Параметр | Описание |
|----------|----------|
| `uncensored` | Разрешает использование нецензурной лексики в ответах |
| `horny` | Активирует откровенные/пошлые ответов от модели |
| `humit-v1` | Делает ответы более человечными используя некоторые дефекты человека (Скоро выйдет "humit-v2") |
| `noEmoji` | Удаляет эмодзи из ответов модели |
| `roleplay` | Включает РП-режим |

## ℹ️ Краткие описания файлов

## ☑️ Распространенные вопросы и проблемы

### Как применить конфигурацию?

1. Скопируйте строку с параметрами, например: `{noEmoji=allow},{horny=allow}`
2. Вставьте её в диалог с моделей, поддерживающей чтение ссылок
3. Модель должна подтвердить загрузку faskFix и начать работу в заданном режиме
```Пример
https://raw.githubusercontent.com/nixitos/faskFix/refs/heads/main/main-prompt.txt //Сама ссылка (Она обязательна)

{noEmoji=allow},{humit-v1=allow},{uncensored},{horny=allow},{roleplay=deny} //Конфигурация
```
_Проверялось на нейросети DeepSeek™_

### Почему модель не отвечает в нужном тоне?

- Убедитесь, что параметр который вам нужен, используется с `allow`
- Проверьте, что вы правильно отправили конфигурацию взятую с сайта
- Некоторые модели могут игнорировать отдельные параметры из-за встроенных ограничений

### Можно ли комбинировать параметры?

Верно, параметры можно соединять друг с другом, они независимы друг от друга. Примеры:
```
{horny=allow},{noEmoji=allow} — пошлый стиль, без эмодзи
{humit-v1=allow},{uncensored} — человеческие ответы с возможным матом
```
### Что делать, если модель упоминает промпт?

По умолчанию faskFix запрещает модели упоминать данный промпт. Если это произошло — попросите её следовать инструкции и больше не упоминать системный промпт.
